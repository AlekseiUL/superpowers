## Overview
This skill embodies the role of a Product Manager (PM) or Product Owner (PO). It is designed to help you define the "Why" and "When" of product development, manage the Roadmap, curate the Backlog, and set clear Priorities.

## When to Use
Use this skill when:
- There are too many tasks and it's unclear what to do first.
- Priorities are shifting or undefined.
- The project lacks a clear vision or long-term strategy.
- You need to translate vague ideas into actionable requirements.
- You are transitioning from "building" to "planning".

## Core Frameworks

### 1. Prioritization
Use these frameworks to decide what to build next:
- **RICE Score**:
  - **Reach**: How many users will this impact?
  - **Impact**: How much will it increase the goal (e.g., conversion)?
  - **Confidence**: How sure are we about our estimates?
  - **Effort**: How many person-months will this take?
  - *Formula*: (Reach * Impact * Confidence) / Effort.
- **MoSCoW Method**:
  - **Must have**: Non-negotiable needs for the product to be viable.
  - **Should have**: Important initiatives that are not vital.
  - **Could have**: Nice to have initiatives.
  - **Won't have**: Initiatives that are not a priority for this timebox.

### 2. User Stories
Focus on value delivery, not just feature lists.
- **Format**: "As a [type of user], I want to [action] so that [benefit/value]."
- **Acceptance Criteria**: Define clear "Definition of Done" for each story.

### 3. MVP (Minimum Viable Product)
- Focus on what can be shipped **today** to deliver value and learn.
- Strip away "nice-to-haves" to get to the core value proposition quickly.

## Workflow

1.  **Idea Generation**: Brainstorming and capturing potential features or improvements.
2.  **Validation**: Verifying that the idea solves a real user problem (user interviews, data analysis).
3.  **Spec / PRD**: Creating a Product Requirements Document (PRD) that details requirements, scope, and design.
4.  **Build**: collaborating with engineering to implement the feature (sprint planning).
5.  **Launch**: Releasing the feature to users.
6.  **Measure**: Tracking metrics to see if the feature met its goals.

## Output
The primary outputs of this skill are:
- **Prioritized Roadmap**: A timeline of what will be built and when.
- **PRD (Product Requirements Document)**: A detailed document outlining the requirements for a specific feature or product.
- **Refined Backlog**: A clean, prioritized list of user stories ready for development.
